Overview:

What it is:

The Rovatech - Site Builder is an extremely felxible and scalable mini-framework for developers and design teams without the hassle of a full blown CMS.

For teams The framework separates structure, functionality, and content. This means that an individual with little to no knowledge of HTML will be able to update content with out much learning and that Designers can update structure and content without messing with the functionality.

For developers this provides a high level of organization with all of the global settings and variable for the web pages stored in PHP scripts. This will also give you peace of mind knowing that your PHP is not getting edited accidentally by other team members.

How it works:

This table illustrates the functionality of this page on the server and how it is built. This is but one very simple example of to provide you the basics of working in the Rovatech - Site Builder framework. This can be used just as easily for a large corporate or ecommerce site.

FileComment
config.php This is where Paths are mapped and global variables are set for item like the image directory, the doctype and database credentials.
settings.php Contains links for CSS and JavaScript file, etc to be available to all pages on your site.
load.php A single file to rule all of your php includes.
web_page.php (class) This class is a template. It return the basic structure for web pages.
page_maker.php Found in the includes folder, this file builds HTML and displays the final page in the browser.
index.php All pages built with teh Rovatech - Site Builder start with a file like this. In this file all of the content file are specified, and Global setting are are either used, overridden or expanded upon.
Header, Nav, home, footer These HTML files contain only the inner elements of their parrent tag. In other words just the content is presented in these files and they can be applied to all pages on the site or use mutiple headers depending on your needs
layout.css Use as mny CSS files as you like, but this is the only one needed for this page.
